git特性：1.直接记录快照（类似于备份），而非差异比较 2.近乎所有操作都是本地操作 git管理的区域分为：1.工作区、2.暂存区、3.git仓库 git的三种状态：1.已修改 2.已暂存 3.已提交 在文件的根目录下执行 git init指令就是将这个文件夹初始化一个git仓库 git status -s:查看文件是否被追踪 绿色表示被追踪 红色相反 git add 文件名：追踪该文件、把已跟踪的且已修改的文件放到暂存区、把有冲突的文件标记为已解决状态[红色的M是已修改未被放入到暂存区，绿的M是已修改并且放入了暂存区] git add .:将所有修改和新增的文件加入暂存区 git commit -m "描述文本":提交到git仓库 git commit -a -m "描述文本":跳过暂存区提交到git仓库 工作区————git仓库 git checkout -- 文件名：撤销对某文件的修改，风险性比较高，需要谨慎使用 git reset HEAD 要移除的文件名称：将文件从暂存区移除 git rm -f 文件名：从仓库中和工作区中都移除文件 git rm --cached 文件名：从仓库中移除文件，保留工作区的文件
